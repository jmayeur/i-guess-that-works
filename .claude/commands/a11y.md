Run a WCAG 2.1 Level AA accessibility audit on this Hugo site using axe-core browser scanning.

Usage: `/a11y [path]`
- With `path` argument: scan only the specified content file (e.g. `/a11y content/posts/2026-06/my-post.md`)
- Without argument: scan only posts changed since the last scan (tracked in `.a11y-last-scan-commit`), plus the homepage
- If no last-scan file exists and no argument given, scan the homepage plus the 5 most recently modified posts

## Triggers
- User says "run accessibility scan", "a11y audit", "check accessibility", "WCAG compliance"
- User references "/accessibility" or "/a11y"

## Procedure

### Step 1: Determine pages to scan

**If `$ARGUMENTS` is provided:**
- Treat it as a `content/` file path (e.g. `content/posts/2026-06/my-post.md`)
- Derive the URL path by stripping `content/` prefix and `.md` suffix (e.g. `/posts/2026-06/my-post/`)
- Scan: homepage `/` + that URL path only

**If no argument:**
- Check for `.a11y-last-scan-commit` in the repo root:
  ```bash
  cat .a11y-last-scan-commit 2>/dev/null
  ```
- If the file exists, get the commit hash from it and find changed content files since then:
  ```bash
  git diff --name-only <last-commit> HEAD -- 'content/posts/**'
  ```
- Convert each changed file path to a URL path (strip `content/` prefix and `.md` suffix, add trailing `/`)
- Scan: homepage `/` + all changed post URLs (up to 10)
- If no posts changed since last scan, report "No content changes since last scan — skipping" and exit without updating the tracking file
- If `.a11y-last-scan-commit` does not exist, scan: homepage `/` + the 5 most recently modified posts:
  ```bash
  find content/posts -name '*.md' | xargs ls -t | head -5
  ```

### Step 2: Start the Hugo dev server
```bash
hugo server --port 1313 &
```
Wait for it to respond with HTTP 200 on `http://localhost:1313`.

### Step 3: Inject axe-core and scan each page
For each page, use browser automation to:
1. Navigate to the page (e.g. `http://localhost:1313/posts/2026-06/my-post/`)
2. Wait 2–3 seconds for content to load
3. Inject and run axe-core:
```javascript
(async () => {
  if (!window.axe) {
    const script = document.createElement('script');
    script.src = 'https://cdnjs.cloudflare.com/ajax/libs/axe-core/4.8.4/axe.min.js';
    document.head.appendChild(script);
    await new Promise(resolve => { script.onload = resolve; });
  }
  const results = await axe.run(document, {
    runOnly: ['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa', 'best-practice']
  });
  window.__axeResults = {
    page: window.location.pathname,
    violations: results.violations.length,
    passes: results.passes.length,
    incomplete: results.incomplete.length,
    details: results.violations.map(v => ({
      id: v.id,
      impact: v.impact,
      description: v.description,
      helpUrl: v.helpUrl,
      nodes: v.nodes.length,
      elements: v.nodes.slice(0, 5).map(n => ({
        html: n.html.substring(0, 200),
        target: n.target,
        failureSummary: n.failureSummary
      }))
    }))
  };
})().then(() => console.log('AXE_SCAN_DONE'));
```
4. Wait 3 seconds, then retrieve `window.__axeResults`.

### Step 4: Kill the dev server
After all pages are scanned, stop the Hugo dev server process.

### Step 5: Update the scan tracking file
Write the current HEAD commit hash to `.a11y-last-scan-commit`:
```bash
git rev-parse HEAD > .a11y-last-scan-commit
```
This file is intentionally not committed — it tracks the last scan state locally.

### Step 6: Generate report
Format results as:

```
## Accessibility Audit Results

**Scan scope:** [argument path | changed since <short-hash> | initial scan]
**Standards:** WCAG 2.1 Level AA + Best Practices
**Pages scanned:** [count]

| Page | Violations | Passes | Worst Impact |
|------|-----------|--------|--------------|
| /    | 2         | 38     | moderate     |

### Violations Found

#### 1. [rule-id] — [impact]
**Description:** [what the rule checks]
**Pages affected:** [list]
**Elements:** [count] instances
**Example:**
`<element that failed>`
**Fix:** [specific recommendation]
**WCAG:** [reference link]

---

### Summary
- Total violations: [n]
- Critical: [n] | Serious: [n] | Moderate: [n] | Minor: [n]

```

## Common Fixes for Hugo Sites

| Violation | Fix |
|-----------|-----|
| `region` (content not in landmarks) | Wrap page content in `<main>`, nav in `<nav>`, footer in `<footer>` — edit the theme layout in `themes/hugo-clarity/layouts/` |
| `heading-order` (skipped heading levels) | Fix heading hierarchy in post content or theme partials |
| `color-contrast` | Adjust colors in theme CSS to meet 4.5:1 ratio (3:1 for large text) |
| `image-alt` | Add descriptive `alt` text in figure shortcodes; use `alt=""` for decorative images |
| `link-name` | Add descriptive text to links; avoid bare "click here" or "read more" |
| `aria-*` | Fix invalid ARIA attributes in theme HTML templates |

## Rules
- Do not modify theme files unless the user explicitly asks for fixes.
- Report findings only; do not auto-fix unless asked.
- If the dev server fails to start, report the error and stop.
- Always update `.a11y-last-scan-commit` after a successful scan (except when exiting early due to no changes).
